Service API
Standardization
Should into Must

Every modern software consumes external services, whether it is a database, email service or logging service. In order to support automation for deployment of multi cloud apps - standardization is no longer a "should", but a "must". Ocopea welcomes such initiatives. One noteworthy example is the Open Service Broker API.
