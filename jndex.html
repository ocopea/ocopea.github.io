Ocopea - pronounced Oh Copy! is an open source project started at Dell EMC.
Ocopea enhances Kubernetes and Cloudfoundry platforms with the concept of application copies, providing developers with easy access to fresh application states.

Extensible

ABOUT

Modern platforms like Kubernetes or Cloudfoundry enable organizations to unleash their creativity and innovate with modern apps often referred as "cloud native apps". Cloud native architecture has not only changed the technology we use, but also how the business interact with software, making it agile, effective and even fun.

Besides benefits, Cloud native apps with their microservices architecture have created a new set of complexities. One example is the ability to for a developer to reproduce an end to end environment consisted of multiple microservices and data stores. Developers struggle with a plethora of tools and manual processes in order to retrieve high quality application copies while trying to debug or run automated tests. With classic monolith apps, using the database copies solves that concern, but cloud native has more data sources with state like the service configuration, orchestrator metadata and each microservice datastore(s).

Similar to other technologies, taking care of those concerns is lagging behind the innovation. With Ocopea we are aiming not only to catch up with traditional solutions, but to harness the advantages of cloud native, taking cloud native programming to extreme.
